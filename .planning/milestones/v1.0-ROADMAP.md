# Milestone v1.0: Public Launch

**Status:** ✅ SHIPPED 2026-03-21
**Phases:** 1–4
**Total Plans:** 14

## Overview

SafeLink India v1.0 — ultra-light emergency information site serving 10,580 static HTML pages across 10 Indian languages, covering all 36 states/UTs, 786 districts, and 225 major cities. Features live NDMA SACHET alert banner via Cloudflare Worker, PWA offline support, and full SEO with hreflang and sitemap.

## Milestone Summary

**Key Decisions:**

- Static HTML over Next.js — matches safe-now.live's 2G-first philosophy; no JS overhead (✓ Validated)
- Cloudflare Pages + Workers — edge-cached globally, free tier, Workers handle live alert API (✓ Validated)
- All 10 languages at launch — India's linguistic diversity is core to accessibility mission (✓ Validated)
- Hybrid static + live alerts — static pages fast and offline-capable; live banner adds real-time value (✓ Validated)
- Direct upload (wrangler) not Git integration — decouples deployment from repo (BUILD: DEPLOY-01)
- 20KB fatal limit for non-English pages — Indic UTF-8 text is ~3× larger than English (I18N-04)
- All-India SACHET RSS feed — state-specific feeds return 404 for multi-word state names (ALERT-01)
- Stale-while-revalidate for SW — balance offline capability and content freshness (PWA-02)

**Issues Resolved:**

- Jinja2 `dict.items()` method collision in templates — fixed with bracket notation `section['items']`
- System git Xcode license block — worked around with `DEVELOPER_DIR=/Library/Developer/CommandLineTools`
- Placeholder KV namespace ID in wrangler.toml — fixed with real namespace f7829bd4ace8419f841a46e52ac1bee5
- Malayalam UTF-8 garbled characters in translation file — fixed before commit

**Tech Debt Deferred to v2:**

- deploy.sh _headers missing `/api/alerts Cache-Control: no-store` — CF CDN may cache Worker responses up to 1 hour
- District/city names render in English on non-English pages — name_translations covers only state names
- `copy_static_files()` in build.py only copies top-level files from static/ — subdirectories silently skipped
- SW only precaches / (homepage) on install — state pages not pre-cached for offline until visited
- manifest.json `lang` field hardcoded `'en'` — non-English PWA installs get English-labelled app
