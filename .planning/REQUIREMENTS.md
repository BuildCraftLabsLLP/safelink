# Requirements: SafeLink India

**Defined:** 2026-03-12
**Core Value:** Anyone in India can reach critical emergency information within seconds on any connection, in their own language.

---

## v1 Requirements

### Navigation & Structure

- [x] **NAV-01**: Homepage displays national emergency numbers (112, 100, 101, 108, NDRF +91-9711077372)
- [x] **NAV-02**: Homepage lists all 36 states + UTs with links to state pages
- [x] **NAV-03**: State page exists for each of the 36 states + UTs with state-specific helplines
- [x] **NAV-04**: District pages exist for all ~750 districts with local district collector helpline, nearest hospital, and NDRF battalion info
- [x] **NAV-05**: City pages exist for major cities (top ~200) with city-specific info
- [x] **NAV-06**: Language selector accessible from every page linking to /hi/, /ta/, /te/, /bn/, /mr/, /kn/, /ml/, /gu/, /pa/ equivalents

### Content — Emergency Guidance

- [x] **CONT-01**: Disaster quick-reference guide page for cyclone (India-specific: track Bay of Bengal/Arabian Sea, pucca building shelter, evacuation zones)
- [x] **CONT-02**: Disaster quick-reference guide page for flood (never wade flood water, crocodile/snake risk, drinking water safety)
- [x] **CONT-03**: Disaster quick-reference guide page for earthquake (drop-cover-hold, aftershock prep, RCC vs load-bearing buildings)
- [x] **CONT-04**: Disaster quick-reference guide page for heatwave (hydration, cooling centres, heat stroke signs — April-June peak)
- [x] **CONT-05**: Disaster quick-reference guide page for landslide (risk zones: NE India, Himachal, Uttarakhand, Western Ghats)
- [x] **CONT-06**: Emergency kit checklist adapted for India (ORS, Aadhaar copy, ration card, AIR battery radio, ₹100/200 notes, N95 masks)
- [x] **CONT-07**: Recovery guidance page (re-entry checklist, snake/scorpion warning after flood, food safety, mold, contractor scam avoidance)
- [x] **CONT-08**: Quick-reference box on homepage/state pages: earthquake (drop/cover/hold), cyclone, flood, fire, gas leak, active threat
- [x] **CONT-09**: First aid reference page (CPR, bleeding, burns, snake bite — India-specific)

### Content — Government Aid

- [x] **AID-01**: Links to PM CARES Fund on relevant pages
- [x] **AID-02**: Per-state CM Relief Fund link on each state page
- [x] **AID-03**: NDRF contact info (national helpline + battalion locations by region) on state pages
- [x] **AID-04**: NDMA SACHET link + direct helpline (1800-11-0551) on homepage and state pages
- [x] **AID-05**: Mental health helplines: iCall (+91-9152987821), Vandrevala Foundation (1860-2662-345), iCall

### Live Alerts

- [x] **ALERT-01**: Active alert banner displayed on homepage and state pages showing NDMA SACHET + IMD alerts
- [x] **ALERT-02**: Alert banner fetches from Cloudflare Worker endpoint (/api/alerts?state={code}) which caches NDMA + IMD feeds (15-min TTL)
- [x] **ALERT-03**: Alert banner fails gracefully — if Worker/API is down, no banner shown (page still works fully)
- [x] **ALERT-04**: Alert banner shows timestamp of last update ("Alerts as of: [time]")

### Internationalisation

- [x] **I18N-01**: All page content available in English (default, no URL prefix)
- [x] **I18N-02**: All page content available in Hindi (/hi/ URL prefix)
- [x] **I18N-03**: All page content available in Tamil (/ta/)
- [x] **I18N-04**: All page content available in Telugu (/te/)
- [x] **I18N-05**: All page content available in Bengali (/bn/)
- [x] **I18N-06**: All page content available in Marathi (/mr/)
- [x] **I18N-07**: All page content available in Kannada (/kn/)
- [x] **I18N-08**: All page content available in Malayalam (/ml/)
- [x] **I18N-09**: All page content available in Gujarati (/gu/)
- [x] **I18N-10**: All page content available in Punjabi (/pa/)
- [x] **I18N-11**: Phone numbers always rendered in ASCII digits regardless of page language
- [x] **I18N-12**: Correct `lang` attribute set on `<html>` for each language page (critical for screen readers + browser rendering)

### Performance & Technical

- [x] **PERF-01**: Every HTML page stays under 15KB (no external CSS, no JS bundles, inline styles only)
- [x] **PERF-02**: Pages use system fonts only — no Google Fonts or @font-face (all 10 scripts supported by default system fonts)
- [x] **PERF-03**: `line-height: 1.6` minimum on all pages (Devanagari/Tamil/Malayalam vowel mark rendering)
- [x] **TECH-01**: Static HTML generated by Python + Jinja2 build script from JSON data files
- [x] **TECH-02**: Deployed to Cloudflare Pages at safelink.serverlord.in
- [x] **TECH-03**: Cloudflare Worker deployed for /api/alerts endpoint with Cloudflare KV caching
- [x] **TECH-04**: All emergency numbers stored in JSON data files (not hardcoded in templates) for easy updates
- [x] **TECH-05**: "Last verified" date displayed on state pages for contact information

### PWA / Offline

- [x] **PWA-01**: manifest.json present making site installable as a PWA
- [x] **PWA-02**: Service worker caches visited pages for offline use
- [x] **PWA-03**: Stale-while-revalidate strategy for HTML pages (serve cached, update in background)
- [x] **PWA-04**: Network-first strategy for /api/alerts (live data prioritised)
- [x] **PWA-05**: Offline indicator shown when user has no network connection ("⚠ You are offline — information may be outdated")

### SEO & Accessibility

- [x] **SEO-01**: hreflang alternate links added to every page for all 10 language variants
- [x] **SEO-02**: sitemap.xml generated covering all pages
- [x] **SEO-03**: `<meta name="geo.region">` on state/district pages
- [x] **A11Y-01**: Skip-to-content link on every page
- [x] **A11Y-02**: Disclaimer and About pages present

## Out of Scope

| Feature | Reason |
|---------|--------|
| SMS fallback | High infrastructure cost, complex to maintain |
| User accounts / login | Friction during emergency; no personalisation value |
| Push notifications | Complex permission flow, infra overhead |
| Maps / embedded Google Maps | Adds 100KB+, breaks 2G usability |
| Video content | Completely unusable on 2G |
| News aggregator | Low value vs complexity; links to official sources instead |
| Native mobile apps | PWA covers offline use case without app store friction |
| User-generated content | Misinformation risk in emergency context |
