    **JSON structure to preserve (from cyclone.json as example):**
    ```json
    {
      "type": "cyclone",          // DO NOT translate - used as identifier
      "title": "translated...",    // Translate
      "slug": "cyclone",          // DO NOT translate - used in URLs
      "sections": [
        {
          "heading": "translated...",   // Translate
          "items": [
            "translated item 1...",     // Translate each item
            "translated item 2..."
          ]
        }
      ],
      "emergency_actions": [
        "translated action 1..."        // Translate
      ],
      "india_specific": "translated..." // Translate (this field is a string, not array, in some guides)
    }
    ```

    **CRITICAL translation rules:**
    1. ACCURACY IS PARAMOUNT - these are emergency safety instructions. Translate meaning faithfully.
    2. Keep ALL phone numbers as ASCII digits: 112, 100, 101, 108, 1070, +91-9711077372, 1800-11-0551
    3. Keep these untranslated: NDMA, NDRF, SEOC, SDRF, IMD, PM CARES, SACHET, NDMA App, All India Radio, Doordarshan, Aadhaar
    4. Keep URLs untranslated: mausam.imd.gov.in, ndma.gov.in, sachet.ndma.gov.in, pmcares.gov.in
    5. Keep "pucca" and "kutcha" as-is or use local equivalent (most Indian languages understand these terms)
    6. "ORS" (Oral Rehydration Solution) - keep as "ORS" with local equivalent in parentheses if space allows
    7. Use UTF-8 characters directly in JSON (not HTML entities)
    8. `"type"` and `"slug"` fields must remain in English (used as identifiers/URLs)
    9. If `india_specific` is a string (not array), translate the whole string but keep embedded data (numbers, URLs, state names) intact

    **Efficiency approach:** To manage context, you may use a Python script to write translation files programmatically. Create a script `scripts/generate_guide_translations.py` that contains all translations as Python dictionaries and writes them as JSON. This keeps translation data organized and makes regeneration easy. After running the script, delete it or keep it for future use.

    Alternatively, write each JSON file directly if you prefer. The key is: all 45 files must exist and be valid JSON with the correct structure.

    Languages in order of translation: hi (Hindi), ta (Tamil), te (Telugu), bn (Bengali), mr (Marathi), kn (Kannada), ml (Malayalam), gu (Gujarati), pa (Punjabi).
  </action>
  <verify>
    Run: `python3 -c "
import json, os
langs = ['hi','ta','te','bn','mr','kn','ml','gu','pa']
guides = ['cyclone','earthquake','flood','heatwave','landslide']
missing = []
bad_json = []
for lang in langs:
    for guide in guides:
        path = f'data/guides_i18n/{lang}/{guide}.json'
        if not os.path.exists(path):
            missing.append(path)
        else:
            try:
                d = json.load(open(path))
                assert 'sections' in d, f'{path}: missing sections'
                assert 'title' in d, f'{path}: missing title'
            except Exception as e:
                bad_json.append(f'{path}: {e}')
if missing:
    print(f'MISSING {len(missing)} files: {missing[:5]}...')
elif bad_json:
    print(f'BAD JSON: {bad_json[:5]}')
else:
    print(f'OK: {len(langs) * len(guides)} guide translation files verified')
"`
  </verify>
