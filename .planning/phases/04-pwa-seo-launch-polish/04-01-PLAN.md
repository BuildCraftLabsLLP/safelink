**static/sw.js** — vanilla JS service worker (~100 lines):
- `const CACHE_NAME = 'safelink-v1';`
- `const PRECACHE_URLS = ['/'];`
- `const HTML_CACHE_MAX_AGE = 24 * 60 * 60 * 1000;` (24h freshness)
- `const HTML_CACHE_MAX_LIFETIME = 7 * 24 * 60 * 60 * 1000;` (7 day expiry)
- INSTALL event: precache PRECACHE_URLS (fetch each, put in cache with custom response header `sw-fetched-on: <timestamp>`); after cache.addAll(PRECACHE_URLS) completes, call `self.skipWaiting()` so the SW activates immediately on first install without waiting for existing clients to close
- ACTIVATE event: delete all caches except CACHE_NAME; call `clients.claim()`
- FETCH event handler:
  - If request.url contains `/api/`: network-first. Try fetch, on success store in cache. On failure, try cache. Return response or empty JSON `{"alerts":[],"cached":true}` if both fail.
  - If request.mode === 'navigate' (HTML page navigation): stale-while-revalidate. Check cache first. If cached, check `sw-fetched-on` header: if age < HTML_CACHE_MAX_AGE, return cached immediately (no background update). If age >= HTML_CACHE_MAX_AGE, return cached AND kick off background fetch to update cache. If not in cache, fetch from network, cache the response (clone it, add `sw-fetched-on` header in a new Response), return original.
  - All other requests (non-navigate, non-API): fall through to network (no caching for assets since all styles are inline).
- When caching HTML responses: store a new Response with the body and a modified headers object that includes `sw-fetched-on: <Date.now()>`. Use `response.clone()` before reading body.

**static/robots.txt** — exact content:
