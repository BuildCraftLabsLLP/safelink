**Step 1: Add Phase 4 tests to test_pages.js.**

The test script accepts a BASE_URL argument as the first CLI argument (e.g., `node test_pages.js https://safelink-india.pages.dev`). Store it as `const BASE_URL = process.argv[2] || 'https://safelink-india.pages.dev';` at the top of the Phase 4 additions. All new test navigations below MUST use `${BASE_URL}` (not hardcoded pages.dev URLs), so the same test file can target any deployment environment.

Append the following test groups to the existing test suite (do not remove any existing tests):

**Group: PWA Metadata (test against English homepage ${BASE_URL}/)**
- `manifest link present` — `document.querySelector('link[rel="manifest"]')?.href` includes `/manifest.json`
- `theme-color meta present` — `document.querySelector('meta[name="theme-color"]')?.content` equals `#FF9933`
- `SW registration script present` — page HTML contains `serviceWorker` string (use `page.content()`)
- `offline indicator script present` — page HTML contains `_ol` string (use `page.content()`)
- `manifest.json accessible` — navigate to `${BASE_URL}/manifest.json`, check response status 200 and body contains `"SafeLink India"`
- `sw.js accessible` — navigate to `${BASE_URL}/sw.js`, check response status 200 and body contains `CACHE_NAME`

**Group: SEO Metadata**
- `hreflang x-default on homepage` — `document.querySelectorAll('link[rel="alternate"][hreflang]')` length >= 11
- `hreflang en on homepage` — one of the alternate links has `hreflang="en"`
- `hreflang hi on homepage` — one of the alternate links has `hreflang="hi"` and href includes `/hi/`
- `canonical link on homepage` — `document.querySelector('link[rel="canonical"]')?.href` starts with `https://safelink.serverlord.in` or `https://safelink-india.pages.dev`
- `geo.region on state page` — navigate to `${BASE_URL}/state/maharashtra/`, check `document.querySelector('meta[name="geo.region"]')?.content` equals `IN-MH`
- `geo.region on district page` — navigate to `${BASE_URL}/state/maharashtra/district/pune/`, check geo.region meta exists and content starts with `IN-`
- `geo.region absent on homepage` — on homepage, `document.querySelector('meta[name="geo.region"]')` is null
- `sitemap.xml accessible` — navigate to `${BASE_URL}/sitemap.xml`, check response status 200 and body contains `<urlset` and `safelink.serverlord.in`
- `robots.txt accessible` — navigate to `${BASE_URL}/robots.txt`, check response 200 and body contains `sitemap.xml`

**Group: PWA - Non-English pages**
- `hreflang on Hindi homepage` — navigate to `${BASE_URL}/hi/`, check `querySelectorAll('link[rel="alternate"][hreflang]').length` >= 11
- `manifest link on Hindi homepage` — `document.querySelector('link[rel="manifest"]')` exists

Keep test structure consistent with existing tests (use the same pass/fail reporting pattern already in test_pages.js).
